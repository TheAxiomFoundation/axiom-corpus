## [0.1.0] - 2026-07-17

### Breaking changes

- Ingest manifests now authorize changed corpus artifacts only when they attest a clean repository-relative generator state with a full commit that is an ancestor of the guarded head; legacy or dirty provenance must be regenerated rather than preserved for compatibility.
- Replace mutable, activate-before-validate corpus publication with one immutable named-release boundary: content-addressed R2 staging and exact readback, direct per-scope provision/navigation counts, post-readback deep validation, Ed25519 release objects, and transactional pointer activation. The signed object `content_sha256` is the release identity; selector-only hashes are provenance. Supabase loads no longer publish scopes or synthesize missing parents, historical versions coexist, and `current.json`/per-scope publish compatibility is removed.
