Added a signed corpus release-manifest surface (`axiom-corpus-release emit-release-manifest` / `verify-release-manifest`, and the `scripts/release_manifest.py` wrapper). The manifest records SHA-256 and row counts for every provisions/inventory/coverage/sources/manifests artifact under `data/corpus`, plus the tracked `claims` JSONL and `DATA_INVENTORY.md`, with declared R2 object keys for corpus artifacts and `created_at` taken from the `HEAD` commit time. It is signed with HMAC-SHA256 over canonical JSON using `AXIOM_CORPUS_RELEASE_SIGNING_KEY`, matching the axiom-encode apply-manifest convention. The first release, `releases/r0/release_manifest.json`, is committed unsigned (hashes are authoritative; sign in place once the release key is provisioned).
