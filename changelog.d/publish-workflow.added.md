Added an automated corpus publication pipeline (`.github/workflows/publish.yml`). On every push to main that touches `data/corpus/**`, it determines which provision versions are unpublished by diffing the git checkout against live `corpus.release_scopes`, then syncs each to R2, loads it into Supabase (auto-registering an active release scope), verifies the per-scope provision counts, and commits a `data/corpus/snapshots/` counts snapshot with `[skip ci]`. A companion staleness guard (`scripts/check_publication_staleness.py`) fails the build if any version has sat on main unpublished for more than 24h, so a stuck publisher can no longer fail silently. Publication driver: `scripts/publish_corpus.py`.
