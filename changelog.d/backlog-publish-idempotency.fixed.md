Made backlog corpus publication idempotent and safe over an arbitrary git range, fixing the three failure classes that broke 59 of 80 scopes in a `workflow_dispatch` backlog run. `load-supabase` gains `--preserve-existing-ids` (upsert a reused `citation_path` onto its existing row instead of inserting a colliding one — the 23505 `UNIQUE(citation_path)` failure), `--synthesize-missing-parents` (backfill a minimal container row for any referenced parent that was never ingested, fixing the 23503 parent_id foreign-key failure), and `--skip-superseded` (never downgrade a citation path already held by a strictly-newer version); the publisher now passes all three. Date columns (`expression_date`, `source_as_of`) are coerced to a valid date or null, so a version-slug-contaminated value can no longer abort a whole scope with 22023 "time zone not recognized". A wholly-superseded version is registered as an inactive release-scope tombstone. The staleness guard now distinguishes never-published scopes (no release-scope row — alarms regardless of the `--since` cutoff) from staged/superseded drift, closing the hole where a too-recent activation cutoff silently hid committed-but-never-published scopes.
