Fixed push-triggered Publish-corpus runs turning red at the final counts-snapshot step even when the publish itself succeeded (`PUBLISHED:1 FAILED:0`). `main` is a protected branch, so the bot's direct `git push` of the snapshot was GH006-rejected, and with no PAT a bot PR cannot trigger the required status checks to auto-merge. The snapshot now lands on the dedicated, unprotected `corpus-snapshots` branch (carrying prior snapshots forward so it accumulates history), and the step is `continue-on-error`, so a snapshot-push problem can no longer red a successful publish — while the publish step itself still fails the run on any bad PUBLISHED/FAILED numbers.
